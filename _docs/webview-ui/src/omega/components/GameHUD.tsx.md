## Overview

This file exports a single presentational React functional component, GameHUD, typed to accept an OmegaSystemsState prop (types imported from ../hooks/useOmegaSystems.js). GameHUD performs pure calculations and renders several HUD sections: team level and an XP bar, a compact tier summary using TIER_CONFIGS, up to three active quests with per-quest progress bars, active mods with optional expiration countdowns, and a vertical stack of toast messages.

A small internal helper ToastItem({ toast }: { toast: ToastMessage }) computes a fade-out opacity from Date.now() - toast.timestamp (fully visible for 2000ms, then linearly fading over 1000ms). The component uses inline React.CSSProperties objects for styling and CSS transitions for animated widths and opacity. It is strictly presentational (no side effects or internal state); time-based values (toast opacity, mod remaining seconds) are recomputed on render, so a parent must re-render periodically for live countdowns.

A parent component obtains an OmegaSystemsState (e.g., from useOmegaSystems) and renders <GameHUD state={state} />. GameHUD displays team level, XP bar, tier summary, active quests (up to three), active mods with optional remaining seconds, and fading toast messages computed from toast timestamps.

## Maintenance Notes

- Performance: Consider memoizing derived values (tier counts, filtered quests) for large lists to avoid repeated work on every render.
- Edge cases: Guard against quest.target === 0 when computing progress to avoid NaN/Infinity; xpForNext === 0 is already guarded.
- Testing: Cover xpForNext === 0, empty arrays, and mod.expiresAt values in the past and future.
- Behavioral caution: Top-level container sets pointerEvents: 'none' while panels set pointerEvents: 'auto' to allow click-through behavior; be careful when adding interactive controls.
