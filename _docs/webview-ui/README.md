## Overview

This directory holds the webview UI layer for the project. At the top level of docs/webview-ui there are no root files; the actual webview source lives in the src/ subdirectory. The src/ directory contains three root-level source files that implement the webview frontend: App.tsx (the main UI entry file for the webview UI), constants.ts (shared static values used by the webview UI), and index.css (global styles applied to the webview layer). Those files together form the single-entry webview UI surface and its styling.

The directory's role in the larger system is to encapsulate the extension's frontend code that will be rendered inside a webview. The src/ subdirectory groups the UI entry point, shared static values, and global styles so developers can quickly find the main entry (App.tsx), adjust reusable constants (constants.ts), or update global styling (index.css). This separation keeps the webview UI layer focused and isolated from other layers of the project while making it straightforward to locate the files needed to modify behavior or appearance of the webview UI.
